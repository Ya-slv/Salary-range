# Инструкция по запуску и тестированию проекта 💸

Все компоненты проекта (модель машинного обучения CatBoost, база данных вакансий для рекомендаций и локальная LLM Qwen для парсинга файлов резюме) полностью настроены и готовы к запуску. Ниже приведено пошаговое руководство.

---

## 🛠 1. Предварительные требования и файлы моделей

Мы уже проверили и настроили все необходимые файлы моделей на ваших дисках:
1. **CatBoost модель:** `data/model.cbm` (загружена и готова к инференсу).
2. **База данных вакансий для рекомендаций:** `data/vacancies.csv` (готова к подбору навыков).
3. **Локальная LLM для парсинга резюме:** `backend/salary/api/models/qwen2.5-0.5b-instruct-q4_k_m.gguf` (размещена на месте, готова к парсингу PDF/DOCX/TXT).

---

## ⚙️ 2. Запуск Backend (Django REST Framework)

Бэкенд работает внутри виртуального окружения `.venv`, которое находится в корне проекта.

1. **Откройте новый терминал** в корневой директории проекта `/Users/kik8/Desktop/projects/Salary-range`.
2. **Активируйте виртуальное окружение** и перейдите в папку Django проекта:
   ```bash
   source .venv/bin/activate
   cd backend/salary
   ```
3. **Запустите Django сервер разработчика:**
   ```bash
   python manage.py runserver
   ```
   Сервер бэкенда запустится на порту `8000` и будет доступен по адресу: `http://127.0.0.1:8000`.
   *Swagger документация по API доступна по адресу:* `http://127.0.0.1:8000/api/schema/swagger-ui/`

---

## 💻 3. Запуск Frontend (React + Vite)

Мы создали современный и быстрый React-интерфейс с поддержкой drag-and-drop загрузки файлов резюме и ручного ввода данных.

1. **Откройте второй терминал** в корне проекта.
2. **Перейдите в папку фронтенда:**
   ```bash
   cd frontend/salary-range-hackaton-frontend
   ```
3. **Запустите сервер разработки Vite:**
   ```bash
   npm run dev
   ```
   Фронтенд запустится на порту `5173`. Откройте в браузере: `http://localhost:5173`.

---

## 🧪 4. Запуск Unit-тестов (Pytest)

Все тесты были настроены с использованием `pytest` и `unittest.mock` для изоляции реальных вызовов ML-моделей и баз данных, что обеспечивает их мгновенное выполнение (менее 1 секунды).

1. **Откройте терминал** в корне проекта.
2. **Запустите тесты:**
   ```bash
   .venv/bin/pytest backend/salary
   ```
   *Все тесты успешно проходят и возвращают зеленый статус!*

---

## 🧠 5. Обучение новой ML-модели

Если вы захотите переобучить модель на новом датасете:
1. Замените или дополните файл вакансий в `data/vacancies.csv`.
2. Запустите скрипт обучения из корня проекта:
   ```bash
   .venv/bin/python train_fork_model.py
   ```
   Скрипт автоматически обучит модель CatBoostRegressor на прогнозирование нижней и верхней границ зарплатной вилки и сохранит готовую модель в `data/model.cbm`.
